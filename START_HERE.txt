HOW TO RUN IT
=============

FASTEST WAY (5 minutes):

1. Open terminal and go to backend directory:
   cd /sessions/determined-keen-wright/pharmalogix/backend

2. Install Python dependencies:
   pip install -r requirements.txt

3. Set your OpenAI API key (required):
   export OPENAI_API_KEY="sk-your-api-key-here"
   (Get a key from https://platform.openai.com/api-keys)

4. Start the server:
   python main.py

5. Open http://localhost:8000 in your web browser

6. Register a new account:
   - Company name: Test Company (or your company name)
   - Email: test@example.com (or any email)
   - Password: anything

7. Try it out:
   - Go to "Data" tab and upload a CSV or Excel file
   - Go to "Chat" tab and ask questions about your data

That's it! You now have a working AI copilot.

COMMON QUESTIONS
================

Q: How do I get an OpenAI API key?
A: Go to https://platform.openai.com/api-keys, sign up, and create an API key.
   The model uses gpt-4o which requires a paid account with credits.

Q: Can I use it without OpenAI?
A: Not as-is. You could replace the OpenAI integration with another LLM
   (Claude, Llama, etc.) by modifying the /api/chat endpoint in main.py

Q: How many users can it support?
A: SQLite is fine for < 10,000 users. For more, migrate to PostgreSQL.

Q: Can I white-label it?
A: Yes! Edit the system prompt, colors, and company name in the code.
   It's designed to be white-labeled for different pharma companies.

Q: How do I add email functionality?
A: Set up an n8n workflow with a webhook, then add the webhook URL in Settings.
   The app will send emails through n8n when you request them.

Q: Is it secure?
A: Yes. It uses JWT tokens, bcrypt hashing, multi-tenant isolation,
   parameterized SQL, and input validation. For production, enable HTTPS.

Q: Can I use it with my own data?
A: Yes! Upload CSV or Excel files with your logistics data, then ask
   questions about it. The AI will analyze your actual data.

TROUBLESHOOTING
===============

Error: "OpenAI API key not configured"
Solution: Set OPENAI_API_KEY environment variable with your actual key

Error: "Database is locked"
Solution: Close other connections. For production, use PostgreSQL instead of SQLite

Error: "Port 8000 already in use"
Solution: Either kill the process using port 8000, or change PORT env var:
         export PORT=8001

Error: "File upload fails"
Solution: Check file is CSV or Excel, not too large (tested up to 10MB)

No messages appear in chat:
Solution: Make sure you uploaded a file first, or check that OpenAI API key is valid

TOKEN EXPIRATION
================

JWT tokens expire after 24 hours. This is fine for the web app since you
just log in again. For long-running applications using the API, you'll need
to implement refresh tokens (not included, can be added).

FILE UPLOAD LIMITS
==================

Tested with files up to 10 MB. SQLite works fine for typical logistics data
(thousands of rows). For larger datasets, consider:
- PostgreSQL for database
- AWS S3 for file storage
- Vector database for embeddings (if needed)

SCALING
=======

Single server: Works fine for up to 10,000 companies
Multiple servers: Use PostgreSQL, Redis for sessions, and load balancing
Large scale: Consider:
- PostgreSQL with read replicas
- Redis for caching
- S3 for file storage
- Elasticsearch for logging
- Monitoring/alerting system
